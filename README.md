# 金庸武侠DND肉鸽踢门团

一个给 AstrBot 使用的群聊文字肉鸽插件。玩家选择金庸武侠门派开局，在武神塔中逐层踢门，遇到战斗、宝箱、奇遇、陷阱、商人等随机事件，通过公开 d20 检定推进流程，最终挑战第 7 层武神殿的武神镜像。

插件目标是把 DND5e 的“公开骰子检定”和金庸武侠门派成长结合成适合群聊游玩的轻量跑团小游戏。

## 功能特性

- 16 大门派开局
- 普通/困难两档难度
- 1~6 层武神塔，每层 3 个随机事件门
- 第 7 层武神殿挑战武神镜像
- 标准 d20 检定，骰点、加值、DC 全部明示
- 所有 d20、d100、伤害骰、掉落概率和随机事件均由代码骰子工具实时结算，不交给 AI 口算
- 每次游戏回复都会按当前场景给出 2~4 个下一步可用中文指令，避免玩家反复查表
- 本门奇遇、残页、补给、碎银、素材、全局奖励
- 钓鱼模块：每层一次，自动鱼池、饵剂修正、鱼获消耗品
- 背囊系统：20/32/50/100 格容量，物品堆叠、丢弃、拾取、使用均按表结算
- 装备系统：武器、防具、饰品按数据表掉落、占格、装备并参与回合制战斗
- 全门派进阶线：16 个门派均有本门奇遇二选一进阶武学，互斥锁定
- 局外强化：通关资源可强化钓鱼准备和初始背囊容量
- 通关后角色飞升冻结
- 游戏结算无第三方 Python 依赖，使用 JSON 保存玩家档案
- 场景小卡片 UI：结合 `astrbot_plugin_html_render` 的显式 `<render>` 逻辑，把回复渲染成聊天窗友好的窄图卡片

## 安装

进入 AstrBot 根目录，将本仓库放入 `data/plugins/`：

```bash
git clone https://github.com/Rudyy898/astrbot_plugin_jinyong_rogue.git data/plugins/astrbot_plugin_jinyong_rogue
```

重启 AstrBot 后启用插件。

### 图片 UI 依赖

本插件会输出显式 `<render>` 自定义 HTML，由 `astrbot_plugin_html_render` 渲染成图片。建议同时启用 `astrbot_plugin_html_render`，并确保其 Playwright 依赖正常；如果未启用，聊天里会看到原始 `<render>` HTML。

## 指令

| 指令 | 说明 |
| --- | --- |
| `/金庸帮助` | 查看游戏指令 |
| `/金庸门派` | 查看 16 大门派 |
| `/金庸开局 门派 [普通\|困难]` | 创建角色 |
| `/金庸状态` | 查看当前角色 |
| `/金庸踢门` | 开启当前层随机事件门；第 7 层挑战武神镜像 |
| `/金庸下一层` | 清完 3 门后触发本门奇遇并进入下一层 |
| `/金庸钓鱼 [饵剂]` | 每层进行一次钓鱼，默认普通蚯蚓饵 |
| `/金庸钓鱼 列表` | 查看鱼池和饵剂 |
| `/金庸背包` | 查看背囊容量、物品、待拾取与占格规则 |
| `/金庸装备 装备名` | 装备武器、防具或饰品 |
| `/金庸技能 武学名\|自动` | 设置战斗优先使用的武学 |
| `/金庸买包` | 遇到商人门后，购买当前商人报价的背囊 |
| `/金庸局外` | 查看局外强化等级与下一阶消耗 |
| `/金庸强化 钓鱼\|背囊\|盘缠\|气血` | 消耗通关后的局外点数提升局外属性 |
| `/金庸丢弃 物品名 [数量]` | 丢弃背包物品 |
| `/金庸拾取 物品名 [数量]` | 拾取因容量不足暂存的物品 |
| `/金庸使用 鱼获名` | 使用鱼获消耗品 |
| `/金庸重置 confirm` | 删除当前角色档案 |

兼容旧指令：以上命令也可继续使用 `/jy` 前缀。

## 游戏规则

### 武神塔

武神塔共 7 层：

- 第 1~6 层：每层固定 3 个事件门
- 第 7 层：武神殿终极挑战
- 清完当前层 3 个门后，使用 `/jy下一层` 触发一次本门奇遇检定
- 第 7 层使用 `/jy踢门` 挑战武神镜像

事件门不会提前生成，只有玩家发送 `/jy踢门` 时才生成并结算。

### 事件类型

| 类型 | 权重 | 说明 |
| --- | ---: | --- |
| 战斗 | 40% | 门派主属性 d20 检定，胜利获得碎银和素材 |
| 宝箱 | 20% | 获得碎银、补给、武学残页，可能触发机关 |
| 奇遇 | 15% | 高人指点，恢复或获得素材 |
| 陷阱 | 15% | 拆解成功得奖励，失败损失 HP |
| 商人 | 10% | 交涉购买补给 |

### 难度

| 难度 | 奇遇 DC | 敌人增幅 | 通关奖励 |
| --- | ---: | ---: | --- |
| 普通 | 10 | +0 | 武道真髓 x2、武学残卷 x5 |
| 困难 | 13 | +3 | 武道真髓 x4、武学残卷 x10 |

### 钓鱼

规则：

- 每层只能钓鱼一次。
- 第 1~6 层默认出现本层山涧鱼池，并有低概率发现隐秘寒潭鱼池。
- 第 7 层 Boss 门口必定出现武神殿门前龙池。
- 钓鱼不进行 d20 成败检定，按鱼池和饵剂提供的 d100 品质修正获得鱼获。
- 局外强化“渔隐行装”会额外提供 d100 品质修正，Lv1/Lv2/Lv3 分别为 +10/+30/+50。
- 局外强化赠送的龙涎香饵会在使用 `/jy钓鱼 龙涎香饵` 时优先消耗，不扣碎银。
- 鱼获作为消耗品进入背包，可通过 `/jy使用 鱼获名` 使用。

饵剂：

- 普通蚯蚓饵
- 秘制香饵
- 龙涎香饵

### 背囊

规则：

- 开局默认装备青布行囊，容量 20 格。
- 鱼获消耗品进入背囊，按数据表的 `stack_size` 堆叠，占用 `ceil(quantity / stack_size) * slot_size` 格。
- 碎银、武学素材、武道真髓、武学残卷、武学残页、补给、小还丹属于资源或旧规则抽象物资，不占背囊格。
- 背囊容量不足时，新鱼获会进入待拾取列表，可先丢弃物品，再使用 `/jy拾取 物品名 [数量]` 收入背囊。
- 背囊只在商人门出售；每次商人只出售当前背囊的下一档，交涉成功报价 9 折。
- 已装备背囊不占自身格数，只能购买更大背囊，不能降级。
- 局外强化“百纳行囊术”只增加开局青布行囊容量，Lv1/Lv2/Lv3 分别为 +4/+8/+12 格。

背囊：

| 背囊 | 容量 | 价格 |
| --- | ---: | ---: |
| 青布行囊 | 20 格 | 0 两 |
| 牛皮百纳囊 | 32 格 | 180 两 |
| 锦缎乾坤囊 | 50 格 | 480 两 |
| 乾坤一气袋 | 100 格 | 1200 两 |

### 局外强化

局外强化保存到 `meta.json`，不会因为 `/jy重置 confirm` 删除角色而清空。只有通关后获得的局外点数可以用于强化，新角色开局会读取同一用户的局外等级。

| 强化 | Lv1 | Lv2 | Lv3 |
| --- | --- | --- | --- |
| 渔隐行装 | 龙涎香饵×1，钓鱼品质+10 | 龙涎香饵×3，钓鱼品质+30 | 龙涎香饵×5，钓鱼品质+50 |
| 百纳行囊术 | 青布行囊+4格 | 青布行囊+8格 | 青布行囊+12格 |
| 行侠盘缠 | 开局碎银+20两 | 开局碎银+50两 | 开局碎银+90两 |
| 护体真气 | 开局HP上限+3 | 开局HP上限+6 | 开局HP上限+10 |

钓鱼和背囊强化从 Lv0 升到 Lv1/Lv2/Lv3，分别消耗武道真髓与武学残卷：`2/5`、`4/10`、`6/15`。盘缠和气血强化消耗较低：`1/3`、`3/8`、`5/12`。

## 数据保存

玩家档案保存到 AstrBot 插件数据目录下：

```text
players.json
```

当前版本按用户 ID 保存角色。角色通关后会进入“飞升冻结”状态，符合蓝本中“单角色通关飞升后档案永久冻结”的设定；如需重开，玩家可手动 `/jy重置 confirm`。

## 数据

插件不依赖外部数据库。游戏结算逻辑不依赖第三方 Python 包；图片 UI 依赖外部插件 `astrbot_plugin_html_render` 负责渲染。
