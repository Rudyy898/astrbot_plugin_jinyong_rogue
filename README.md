# 金庸武侠DND肉鸽踢门团

一个给 AstrBot 使用的群聊文字肉鸽插件。玩家选择金庸武侠门派开局，在武神塔中逐层踢门，遇到战斗、宝箱、奇遇、陷阱、商人等随机事件，通过公开 d20 检定推进流程，最终挑战第 7 层武神殿的武神镜像。

插件目标是把 DND5e 的“公开骰子检定”和金庸武侠门派成长结合成适合群聊游玩的轻量跑团小游戏。

## 功能特性

- 16 大门派开局
- 普通/困难两档难度
- 1~6 层武神塔，每层 3 个随机事件门
- 第 7 层武神殿挑战武神镜像
- 标准 d20 检定，骰点、加值、DC 全部明示
- 所有 d20、d100、伤害骰、掉落概率和随机事件均由代码骰子工具实时结算，不交给 AI 口算
- 每次游戏回复都会按当前场景给出 2~4 个下一步可用中文指令，避免玩家反复查表
- 本门奇遇、残页、药品、碎银、素材、全局奖励
- 钓鱼模块：每层一次，自动鱼池、饵剂修正、鱼获消耗品
- 背囊系统：20/32/50/100 格容量，物品堆叠、丢弃、拾取、使用均按表结算
- 装备系统：武器、防具、饰品按数据表掉落、占格、装备并参与回合制战斗
- 全门派进阶线：16 个门派均有本门奇遇二选一进阶武学，互斥锁定
- 局外强化：通关资源可强化钓鱼准备和初始背囊容量
- 通关后角色飞升冻结
- 使用 JSON 保存玩家档案
- 场景小卡片 UI：插件内置 Pillow 图片渲染，直接发送聊天窗友好的窄图卡片，不依赖 `astrbot_plugin_html_render`

## 安装

进入 AstrBot 根目录，将本仓库放入 `data/plugins/`：

```bash
git clone https://github.com/Rudyy898/astrbot_plugin_jinyong_rogue.git data/plugins/astrbot_plugin_jinyong_rogue
```

重启 AstrBot 后启用插件。

### 图片 UI 依赖

本插件使用 Pillow 在插件内部生成 PNG 小卡片，并直接通过 AstrBot 图片组件发送。无需安装或启用 `astrbot_plugin_html_render`。如果 Pillow 不可用，插件会自动退回纯文本回复。

## 游戏规则

### 武神塔

武神塔共 7 层：

- 第 1~6 层：每层固定 3 个事件门
- 第 7 层：武神殿终极挑战
- 清完当前层 3 个门后，使用 `/jy下一层` 触发一次本门奇遇检定
- 第 7 层使用 `/jy踢门` 挑战武神镜像

事件门不会提前生成，只有玩家发送 `/jy踢门` 时才生成并结算。

### 事件类型

| 类型 | 权重 | 说明 |
| --- | ---: | --- |
| 战斗 | 40% | 门派主属性 d20 检定，胜利获得碎银和素材 |
| 宝箱 | 20% | 获得碎银、药品、武学残页，可能触发机关，也可能获得装备奖励 |
| 奇遇 | 15% | 高人指点，恢复或获得素材 |
| 陷阱 | 15% | 拆解成功得奖励，失败损失 HP |
| 商人 | 10% | 交涉购买药品，并刷新背囊和装备报价；可出售背包占格物品，已装备装备不会出售 |

### 武学领悟

规则：

- 战斗、奇遇和探索会获得武学素材；宝箱、本门奇遇和高层掉落会获得中阶或顶级残页。
- `/jy武学` 会列出本门可领悟武学、当前资源、消耗和不可领悟原因。
- 进阶武学消耗中阶残页 x1 + 武学素材 x3；顶级绝学消耗顶级残页 x1 + 武学素材 x8。
- 中阶残页可用 `/jy合成残页 [数量]` 向上合成为顶级残页，比例为 4:1。
- 通关、战败或放弃结算时，本局剩余中阶残页会与局外库存合并，并自动按 4:1 尝试合成顶级残页。
- 本门顶级残页只解锁本命门派的局外顶级绝学；解锁后，该门派新局可在 `/jy武学` 中领悟。
- 有局外顶级残页或传说残页时，需要手动使用 `/jy解锁绝学 绝学名` 扣除残页并解锁，避免自动解锁到不想要的绝学；用 `/jy绝学` 查看绝学库和技能详情。
- 通关 Boss 后会进行 d100 传说残页掉落检定，传说残页可手动解锁 5 个通用传说绝学之一；通用传说绝学所有门派新局都可领悟。
- 同一互斥武学组只能领悟一门，领悟后可用 `/jy技能 武学名` 设为常用技能。

### 难度

| 难度 | 奇遇 DC | 敌人/MP | 残页奖励 |
| --- | ---: | --- | --- |
| 普通 | 10 | 使用旧困难敌人基线；战斗每回合恢复 1MP，非战斗 2MP | 战斗不掉中阶残页；通关低概率获得中阶残页 |
| 困难 | 14 | 敌人再强一层；战斗每 2 回合恢复 1MP，非战斗 2MP | 战斗概率掉中阶残页；通关必得门派顶级残页，极低概率传说残页 |
| 炼狱 | 16 | 需要对应门派困难通关后才可选择；最高敌人增幅；战斗不恢复 MP，非战斗 1MP | 胜武神镜像后可挑战真·武神，胜利必得传说残页 x1；不挑战则低概率获得 |

战斗中若 MP 为 0 且没有 0MP 攻击招式，只能逃跑；逃跑成功本门完成但不获得战斗奖励，逃跑失败敌人获得优势命中加成。

### 钓鱼

规则：

- 每层只能钓鱼一次。
- 第 1~6 层默认出现本层山涧鱼池，并有低概率发现隐秘寒潭鱼池。
- 第 7 层 Boss 门口必定出现武神殿门前龙池。
- 钓鱼不进行 d20 成败检定，按鱼池和饵剂提供的 d100 品质修正获得鱼获。
- 局外强化“渔隐行装”会额外提供 d100 品质修正，Lv1/Lv2/Lv3 分别为 +10/+30/+50。
- 未指定特殊饵剂时，默认使用普通蚯蚓饵，不消耗碎银或物品。
- 局外强化赠送的龙涎香饵会在使用 `/jy钓鱼 龙涎香饵` 时优先消耗；没有随身饵剂时也不扣碎银。
- 鱼获和药品作为消耗品进入背包，可通过 `/jy使用 名称` 使用。

饵剂：

- 普通蚯蚓饵
- 秘制香饵
- 龙涎香饵

### 背囊

规则：

- 开局默认装备青布行囊，容量 20 格。
- 装备、鱼获和药品消耗品进入背囊，按数据表的 `stack_size` 堆叠，占用 `ceil(quantity / stack_size) * slot_size` 格。
- 碎银、武学素材、武道真髓、武学残页、随身饵剂也显示在背包资源区，但占 0 格。
- 背囊容量不足时，新装备、鱼获或药品会进入待拾取列表，可先出售、丢弃或使用物品，再用 `/jy拾取 物品名 [数量]` 收入背囊。
- 背囊只在商人门出售；每次商人必定出售当前背囊的下一档，交涉成功报价 9 折。
- 商人门会额外刷新一件装备报价；背囊和装备都用 `/jy购买 商品名` 买入，背包中药品、鱼获、未装备装备等占格物品可用 `/jy出售 物品名 [数量]` 卖出换碎银。
- 商人门不会在首次遇到后立刻结束；只要没有使用 `/jy离开商人`，玩家可以先去踢其他门，再回到该门继续购买或出售。
- 若连续两层没有遇到商人门，下一层三道殿门都踢完后会额外出现一名游商；该游商独立于三道殿门之外，只提供购买和出售。
- 已装备背囊不占自身格数，只能购买更大背囊，不能降级。
- 局外强化“百纳行囊术”只增加开局青布行囊容量，Lv1/Lv2/Lv3 分别为 +4/+8/+12 格。

背囊：

| 背囊 | 容量 | 价格 |
| --- | ---: | ---: |
| 青布行囊 | 20 格 | 0 两 |
| 牛皮百纳囊 | 32 格 | 180 两 |
| 锦缎乾坤囊 | 50 格 | 480 两 |
| 乾坤一气袋 | 100 格 | 1200 两 |

### 局外强化

局外强化保存到 `meta.json`，不会因为 `/jy重置 confirm` 删除角色而清空。每局结算获得的武道真髓可以用于强化，新角色开局会读取同一用户的局外等级。

局外资源里的“小还丹”可通过 `/jy小还丹 0|1|2|3` 设置新局携带目标数量。`0` 表示不带，`1` 是默认档；`2`、`3` 需要“还丹护命”升级到对应上限才会生效。开局实际带入数量取局外库存、携带目标、升级上限三者最小值；死亡时自动消耗 1 颗并半血复活。背包中获得的“小还丹”仍是普通回血药品，可用 `/jy使用 小还丹` 回复 HP。

| 强化 | Lv1 | Lv2 | Lv3 |
| --- | --- | --- | --- |
| 渔隐行装 | 龙涎香饵×1，钓鱼品质+10 | 龙涎香饵×3，钓鱼品质+30 | 龙涎香饵×5，钓鱼品质+50 |
| 百纳行囊术 | 青布行囊+4格 | 青布行囊+8格 | 青布行囊+12格 |
| 行侠盘缠 | 开局碎银+20两 | 开局碎银+50两 | 开局碎银+90两 |
| 护体真气 | 开局HP上限+3 | 开局HP上限+6 | 开局HP上限+10 |
| 还丹护命 | 最多携带2颗护命小还丹 | 最多携带3颗护命小还丹 | - |

钓鱼和背囊强化从 Lv0 升到 Lv1/Lv2/Lv3，分别消耗武道真髓：`10`、`18`、`28`。盘缠和气血强化消耗较低：`6`、`12`、`20`。
还丹护命默认 Lv0 即可最多携带 1 颗；升到 Lv1 消耗武道真髓 `20`，最多携带 2 颗；升到 Lv2 消耗武道真髓 `50`，最多携带 3 颗。

## 数据保存

玩家档案保存到 AstrBot 插件数据目录下：

```text
players.json
```

当前版本按用户 ID 保存角色。角色通关后会进入“飞升冻结”状态，符合蓝本中“单角色通关飞升后档案永久冻结”的设定；如需重开，玩家可手动 `/jy重置 confirm`。
